### 2.4 cmd+Q 退出 — **本次修了一个真实 bug**

**修复前问题**:osascript 走 macOS `terminateApp` 直接 SIGTERM 主进程,
**不走 Tauri `RunEvent::ExitRequested`**,原 `shutdown_sidecar` 不触发,
sidecar 变孤儿继续占 port 30000(用户反馈"以为退了,实际还在跑")。

**修法**:

1. `start_sidecar` 拿 child 后把 PID 写入全局 `Lazy<Mutex<Option<u32>>>`
2. `AppState::new` 安装 `libc::atexit` 钩子
3. 主进程任何路径退出(`cmd+Q` / `SIGTERM` / `panic unwinding`)都同步 `SIGKILL` sidecar

**验证**:cmd+Q 后 ps 显示无任何 nexus 进程,port 30000 释放。

```rust
// runtime.rs 核心
static SIDECAR_PID: Lazy<Mutex<Option<u32>>> = Lazy::new(|| Mutex::new(None));

fn kill_sidecar_at_exit() {
    if let Some(pid) = SIDECAR_PID.lock().unwrap().take() {
        unsafe { libc::kill(pid as i32, libc::SIGKILL); }
    }
}
// AppState::new 里 Once::call_once 安装
unsafe { libc::atexit(atexit_handler); }
```
