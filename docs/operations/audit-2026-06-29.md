## 4.1 关键修复（5 项）

### C001 — Gateway `_get_or_create_session` 静默吞错

**现象**：`except Exception: pass` 会让任何 session 解析异常消失，仅返回 None。

**修复**：`nexus/backend/channels/gateway.py`
- `except Exception` 改为 `logger.warning(...)` + 精确捕获
- 嵌套 `_send_error` 加 `except (httpx.RequestError, RuntimeError, OSError)`
- `import httpx` 提前到模块顶部（linter 提示）

### C002 — 微信账号 token 明文 base64 存储

**现象**：`~/.nexus/state/wechat/{id}.json` 里 `token` 字段是 `base64(token)`，明文 base64 不构成加密。

**修复**：`nexus/backend/channels/wechat_account.py` 全量 Fernet 化：
- 新字段 `tokenVersion`：1 = legacy base64，2 = Fernet
- `_resolve_encryption_key()`：env `NEXUS_TOKEN_ENCRYPTION_KEY` 优先，否则从 `~/.nexus/.secret`（0o600 自动生成）读
- `_load_account` 自动迁移：检测旧版 → 解码 → Fernet 重写 → `tokenVersion=2`
- 新增 7 个测试覆盖：round-trip / 自动迁移 / Fernet 损坏 / secret 文件权限 / env 优先 / env 格式错 / chmod 验证

**用户须知**：备份 `~/.nexus/` 时务必含 `.secret`，否则恢复后所有 token 解不开，需重新扫码。

### C007 — Gateway `route_message` 嵌套发送兜底缺失

**现象**：外层 `except` 兜底时 `_send_error` 自身抛错无保护，整个路由崩溃。

**修复**：嵌套捕获 `(httpx.RequestError, RuntimeError, OSError)`，其余异常走 `logger.exception` 留全栈。

### D001+D002 — `QualityGateMiddleware` 写拦截工具白名单不全

**现象**：`_FILE_TOOLS` 只列 `edit_file` / `write_file`，LLM 用 `create_file` / `apply_patch` 等别名直接绕过。

**修复**：`nexus/backend/quality/middleware.py`
- 扩展到 7 个写工具名
- 加 `_WRITE_TOOL_PATTERNS` 黑名单（防白名单漏字）
- 加 `_READ_ONLY_TOOLS` 白名单（更严格的只读工具）
- 加 `_is_write_tool()` 判定函数
- 同步 `wrap_tool_call`（sync）用 ThreadPoolExecutor 包 `awrap_tool_call`，避免同步钩子直接 `asyncio.run` 抛 "running event loop"

### E001 — 前端 WS 钩子双实现无适配

**现象**：`ChatArea.tsx` 同时调 `useTauriWs` + `useWebSocket`，Tauri 内嵌浏览器两个都会触发，重复连接。

**修复**：
- 新增 `frontend/src/hooks/useWsConnection.ts`：根据 `window.__TAURI_INTERNALS__` 自动二选一
- `ChatArea.tsx` 改用 `useWsConnection` 单一钩子

## 4.2 安全加固（4 处）

### A009 — WS token 时序攻击

**修复**：`nexus/backend/main.py`
- `token != expected` → `hmac.compare_digest(token or "", expected)`
- expected 为空时直接拒绝（避免空 secret 误过）

### A008 — CORS 通配过宽

**修复**：
- `allow_methods=["*"]` → `["GET", "POST", "PUT", "DELETE", "OPTIONS"]`
- `allow_headers=["*"]` → `["Authorization", "Content-Type", "X-Requested-With"]`
- `NEXUS_ALLOWED_ORIGINS` 含 `*` 时启动直接 `ValueError`（allow_credentials=True 与 `*` 不兼容）

### B13 — LLM 异常回显泄漏 API key

**修复**：`nexus/backend/llm/errors.py`
- 新增 `_redact_secrets()`：两道正则
  1. 已知前缀：`sk-` / `sk-proj-` / `ghp_` / `xai-` / `anthropic-` + 20+ 位
  2. 兜底：任意 32+ 位 `[A-Za-z0-9_-]`（Bearer token）
- `_build_message` 包装前先 redact
- 新增 `tests/test_llm_error_redaction.py`（9 例）

### D07 — resume_secret 回退到 ws_token

**现象**：`resilience/resume.py` 的 `_get_secret` 在 `resume_secret` 缺失时回退 `ws_token`；但 ws_token 是 WS 端点固定密码，在 DMG 构建里明文可见，断线重连 token 强度被降到同一水平。

**修复**：
- 删除 ws_token 回退分支
- 长度 < 32 字节直接 `RuntimeError`（HMAC 最佳实践）
- 测试 fixture 改 32+ 字节 secret；删除旧 `test_falls_back_to_ws_token_when_no_resume_secret`
